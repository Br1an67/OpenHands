Add typed event methods to AnalyticsService so callers can use `track_conversation_created(conversation_id=..., trigger=..., llm_model=...)` instead of building raw property dicts.

Purpose: Replace the fragile pattern of callers constructing property dicts inline (e.g., auth.py line 268, webhook_router.py line 174) with methods that own their event schema. This prevents typos in property names, makes the event contract explicit, and enables IDE autocomplete.

Output: 10 typed methods on AnalyticsService, one per event constant, with full test coverage. Phase 6 will migrate call sites to use these methods.
</objective>
