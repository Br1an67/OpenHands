**Insights to create (6 tiles):**

1. **Web Vitals -- LCP (Largest Contentful Paint)**

2. **Web Vitals -- FCP (First Contentful Paint)**
   - Type: TRENDS
   - Event: `$web_vitals`
   - Math: `avg` on property `$web_vitals_FCP_value`
   - Filter: `$web_vitals_FCP_event` is set (exists)
   - Display: line chart
   - Date range: last 30 days, daily interval

3. **Web Vitals -- INP (Interaction to Next Paint)**
   - Type: TRENDS
   - Event: `$web_vitals`
   - Math: `avg` on property `$web_vitals_INP_value`
   - Filter: `$web_vitals_INP_event` is set (exists)
   - Display: line chart
   - Date range: last 30 days, daily interval

4. **Web Vitals -- CLS (Cumulative Layout Shift)**
   - Type: TRENDS
   - Event: `$web_vitals`
   - Math: `avg` on property `$web_vitals_CLS_value`
   - Filter: `$web_vitals_CLS_event` is set (exists)
   - Display: line chart
   - Date range: last 30 days, daily interval

5. **JS Error Rate (daily count)**
   - Type: TRENDS
   - Event: `$exception`
   - Math: `total` (count)
   - Display: line chart
   - Date range: last 30 days, daily interval

6. **Top JS Errors (by exception type)**
   - Type: TRENDS
   - Event: `$exception`
   - Math: `total` (count)
   - Breakdown: `$exception_type`
   - Display: table
   - Date range: last 30 days

- Use `requests` library (standard in the project's Python env)
- First create the dashboard (POST /api/projects/{id}/dashboards/)
- Then create each insight (POST /api/projects/{id}/insights/) with `dashboards: [dashboard_id]`
- Print the dashboard URL on success
- Script should be idempotent-safe: if a dashboard named "Frontend Health" already exists, print its URL and exit without creating duplicates
- Exit with code 0 on success, code 1 on failure with error message
