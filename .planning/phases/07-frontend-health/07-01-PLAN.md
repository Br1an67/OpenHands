In `posthog-wrapper.tsx`, add three config options to the `options` prop of `PostHogProvider`:

1. **Web vitals** -- add `capture_performance` with `web_vitals: true` to enable FCP, LCP, INP, CLS capture. Keep `network_timing: true` as well since it enhances session replay with network performance data:
   ```

     web_vitals: true,
   },
   ```

2. **Error tracking** -- add `capture_exceptions: true` to capture unhandled JS errors and promise rejections:
   ```
   capture_exceptions: true,
   ```

3. **Network recording** -- Network payload capture for session replay is controlled via PostHog project settings (not client-side SDK config). However, enabling `network_timing: true` in `capture_performance` (step 1) ensures network timing data appears in session replays. No additional client-side config is needed for network recording -- it's a project-level toggle in PostHog.

The final options object should be:
```typescript
{
  api_host: "https://us.i.posthog.com",
  person_profiles: "identified_only",
  capture_performance: {
    network_timing: true,
    web_vitals: true,
  },
  capture_exceptions: true,
  bootstrap: bootstrapIds,
  __add_tracing_headers: [window.location.hostname],
}
```

Do NOT change any other behavior (bootstrap, api_host, person_profiles, tracing headers).
